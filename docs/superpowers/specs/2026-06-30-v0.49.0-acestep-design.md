# v0.49.0 - ACE-Step text-to-music runtime

**Goal:** Add ACE-Step (ACE-Step/ACE-Step-v1-3.5B) as a new text-to-music
runtime under the EXISTING `audio/generation` modality, served at
`POST /v1/audio/music`, with optional lyrics for full songs and
instrumental-by-default for backing tracks.

**Status:** design locked 2026-06-30 (user-approved). Grounded by the
`understand-translation-acestep` workflow map + live API research.

## Decisions

- **No new modality.** ACE-Step reuses `audio/generation`. New runtime +
  bundled script + small wire additions only.
- **Bundled script** `ace-step-v1-3.5b` (discoverable; `muse pull` installs
  deps + weights). `device: cuda`, GPU-required.
- **`trust_remote_code` NOT needed** - `ACEStepPipeline` is a direct
  package import (`from acestep.pipeline_ace_step import ACEStepPipeline`),
  not an `AutoModel.from_pretrained(..., trust_remote_code=True)` boundary.
- **Install** pins the git source: `acestep @
  git+https://github.com/ace-step/ACE-Step.git` (the PyPI `ace-step` is a
  stale v0.1.0).
- **Lyrics exposed** as an optional `/v1/audio/music` field. Empty/None ->
  instrumental (`[instrumental]`); `[verse]`/`[chorus]` lyrics -> sung song.
- **Duration ceiling raised** 120s -> 240s on the route (ACE-Step does long
  clips; per-model clamp still happens in the runtime against
  `max_duration`).
- `supports_music: true`, `supports_sfx: false` (music-only; `/v1/audio/sfx`
  returns 400 for it).

## ACE-Step SDK contract (from API research)

- Constructor: `ACEStepPipeline(checkpoint_dir=<local_dir>, device_id=0,
  dtype="bfloat16", cpu_offload=, overlapped_decode=, torch_compile=,
  quantized=)`. `checkpoint_dir=None` auto-downloads; we pass muse's
  managed `local_dir`.
- Call: `pipe(format="wav", audio_duration=<s>, prompt=<genre/style tags>,
  lyrics=<structured | "[instrumental]">, infer_step=<steps>,
  guidance_scale=<cfg>, manual_seeds=<seed>, save_path=<tmp.wav>,
  batch_size=1)`.
- **Output: writes audio file(s) to disk and RETURNS file path(s) + a
  params dict** (NOT in-memory arrays, unlike StableAudio). 48kHz stereo.
  The runtime calls with a temp `save_path`, reads the WAV back via
  `soundfile`, and deletes the temp file in a `finally` (leak-safe, per the
  #200 temp-WAV lesson). Return-shape is resolved defensively (returned
  path if on disk, else the `save_path` we provided).
- VRAM ~8GB min with `cpu_offload/overlapped_decode/torch_compile`;
  comfortable on 12GB at bf16. The three low-VRAM flags are exposed as
  optional manifest capabilities (default off).

## Test strategy

Runtime + script tests are **fully mocked** (patch module-level `torch` /
`ACEStepPipeline` sentinels; mock `pipe()` to write a real temp WAV that
`soundfile` reads). The real SDK I/O (return shape, `manual_seeds` format,
instrumental convention) is **verified on the .204 GPU box before the
release tag** (Step B1: real-API verification for a custom-SDK runtime).
