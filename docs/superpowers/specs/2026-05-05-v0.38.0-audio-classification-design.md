# v0.38.0 audio/classification Design

**Date:** 2026-05-05
**Status:** Approved for plan
**Closes:** #102 audio/classification (speaker, emotion, language, VAD)

## Goal

A new `audio/classification` modality with one route `POST /v1/audio/classifications`. Wire shape mirrors `/v1/text/classifications`: list of `{label, score}` pairs per input. One bundled general-purpose model (AST AudioSet 527-class), curated entries for emotion, speech commands, and language ID.

Speaker diarization (frame-level segments) is structurally different from single-label classification and is deferred to a future `audio/diarization` modality.

## Wire shape

`POST /v1/audio/classifications`:

Multipart input:

| field | type | required | notes |
|---|---|---|---|
| `file` | file | yes | WAV/MP3/FLAC/OGG/WebM. Cap via `MUSE_AUDIO_CLS_MAX_BYTES` (default 50MB). |
| `model` | str | no | Modality default applies. |
| `top_k` | int | no | Truncate the per-input label list. Default: return all. |

Response:

```json
{
  "id": "audio-cls-<uuid>",
  "model": "ast-audioset",
  "results": [
    [
      {"label": "Speech", "score": 0.92},
      {"label": "Music", "score": 0.05}
    ]
  ]
}
```

`results` is always a list of lists (per-file inner list). Inner list sorted by score descending.

## Runtime

**`HFAudioClassifier`** wraps `AutoModelForAudioClassification` + `AutoFeatureExtractor`. Audio decoding via librosa (already a dep of audio_embedding / audio_transcription). Per-model resample rate read from `feature_extractor.sampling_rate`.

Inference:
1. `librosa.load(path, sr=feature_extractor.sampling_rate, mono=True)` -> waveform
2. `feature_extractor(waveform, sampling_rate=..., return_tensors="pt")` -> input tensors
3. `model(**inputs).logits` -> (1, N) tensor
4. `softmax(logits)` -> per-class probabilities
5. Build `{label: score}` dict via `model.config.id2label`

Multi-label classifiers (sigmoid heads) are detected via `model.config.problem_type == "multi_label_classification"`; in that case scores come from sigmoid not softmax.

## Bundled (1)

**`ast-audioset`** = `MIT/ast-finetuned-audioset-10-10-0.4593`
- ~340MB, BSD-3-Clause, 527 AudioSet classes (speech / music / animals / vehicles / alarms / household / etc.)
- General-purpose audio event classifier; covers most "what's in this audio?" use cases.
- Joins the smoke matrix.

## Curated (4)

- `wav2vec2-emotion-en` = `ehcalabres/wav2vec2-lg-xlsr-en-speech-emotion-recognition`: English emotion (8 classes, ~1.2GB)
- `hubert-superb-er` = `superb/hubert-base-superb-er`: smaller emotion alternative (~360MB)
- `ast-speech-commands` = `MIT/ast-finetuned-speech-commands-v2`: 35 speech commands (~340MB)
- `mms-lid-126` = `facebook/mms-lid-126`: 126-language spoken language ID (~700MB)

## HF plugin

Sniff `audio-classification` tag. Repo-name fallback for `wav2vec`, `ast-`, `hubert`, `mms-lid`, `panns`. Priority 110 (same slot as audio/embedding, image/cv).

## Out of scope

- Speaker diarization (segments + speaker labels). Different output shape; future `audio/diarization` modality.
- Streaming audio classification (frame-by-frame VAD). One-shot per file in v1.
- Speaker verification / identification (paired-input task). Future `audio/speaker` modality if demand justifies.

## Test strategy

- **Codec**: top_k truncation, OpenAI envelope shape, multi-label vs softmax distinction.
- **Runtime**: mocked AutoModel + AutoFeatureExtractor + librosa; assert id2label resolution, softmax/sigmoid dispatch, sampling-rate forwarding.
- **Route**: FakeModel, multipart upload, OpenAI error envelope on 400/404/500, MUSE_AUDIO_CLS_MAX_BYTES cap.
- **HF plugin**: tag dispatch, repo-name fallbacks, multi_label flag from problem_type.
- **Bundled script**: manifest required keys, capability flags.
- **Smoke matrix**: bundled `ast-audioset` joins.
