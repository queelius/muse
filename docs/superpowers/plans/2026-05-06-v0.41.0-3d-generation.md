- [ ] **A1.** `protocol.py`: `Generation3DResult(glb_bytes: bytes, model_id: str, format: str = "glb")` dataclass + `Generation3DBackend` Protocol declaring `image_to_3d(image_path)` + `text_to_3d(prompt)` methods (both optional via runtime checks; capability flags gate which one is callable).

- [ ] **A2.** `codec.py`: `encode_3d_response(results, model_id, response_format)` returning the OpenAI-shape envelope. `b64_json` mode encodes via `base64.b64encode`. `url` mode writes to a tmp file and returns a `file://` URL.

- [ ] **A3.** `client.py`: `Generation3DClient` with `.from_text(prompt, *, model=None, n=1, ...)` and `.from_image(image, *, model=None, ...)`. Image accepts bytes / file-like / Path. Mirrors `GenerationsClient` (image_generation/client.py) shape.

- [ ] **A4.** `__init__.py`: exports `MODALITY = "3d/generation"`, `Generation3DResult`, `Generation3DClient`, `build_router`. PROBE_DEFAULTS shape mirrors audio_classification (synthetic image → image_to_3d).

- [ ] **A5.** `routes.py`: build_router skeleton with both endpoints registered. Bodies raise `NotImplementedError("Task B will fill")`.

- [ ] **A6.** Tests for protocol, codec, client. Run `pytest tests/modalities/model_3d_generation/ -q` to verify.

- [ ] **A7.** Commit: `feat(3d/generation): protocol + codec + client + skeleton`.

### Task C: TripoSRRuntime (~4h)

**Files:**
- `src/muse/modalities/model_3d_generation/runtimes/__init__.py`
- `src/muse/modalities/model_3d_generation/runtimes/triposr.py`
- `tests/modalities/model_3d_generation/runtimes/test_triposr.py`

- [ ] **C1.** Define `TripoSRRuntime` class. Constructor accepts `model_id`, `hf_repo`, `local_dir`, `device`, `dtype` (matching other muse runtimes). Lazy-import sentinels for `torch`, `tsr` (TripoSR's package), `trimesh`, `PIL`. `_ensure_deps()` populates them; constructor raises clear RuntimeError on missing deps with which package missing.

- [ ] **C2.** `__init__` body: load via `tsr.system.TSR.from_pretrained(local_dir or hf_repo, weight_name=...)`, move to device, set inference mode. Use `LoadTimer` from `muse.core.runtime_helpers`.

- [ ] **C3.** `image_to_3d(image_path)` method: open the PIL image, preprocess via TripoSR's `process_image` helper, call the model's forward, extract mesh via `model.extract_mesh(scene_codes, has_vertex_color=False)[0]`, export the trimesh.Scene to GLB bytes via `mesh.export(file_obj, file_type="glb")`. Returns `[Generation3DResult(glb_bytes=..., model_id=self.model_id)]`.

- [ ] **C4.** No `text_to_3d` method (TripoSR is image-only). Capability flags on the manifest gate the route; the runtime doesn't need to implement the unsupported direction.

- [ ] **C5.** Tests with mocked dependencies: torch, tsr, trimesh, PIL all patched. Verify the deferred-import pattern, the constructor's dep-missing errors, and the inference path returns a `bytes` GLB.

- [ ] **C6.** Run tests; commit: `feat(3d/generation): TripoSRRuntime`.

## Risks recap

- TripoSR's `tsr` PyPI package: pin a known-good version range in pip_extras; document the dependency on its maintainer.
- TRELLIS / Hunyuan3D-2 require `trust_remote_code=True`: deferred (v0.41.x), but the curated yaml entries should set the flag.
- GPU-only modality: smoke matrix without GPU can only verify load-time, not inference quality. Document.
