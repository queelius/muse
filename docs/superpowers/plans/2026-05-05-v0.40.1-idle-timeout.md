> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan.

## Tasks

### Task A: IdleSweeper class + unit tests

**Files:** `src/muse/cli_impl/idle_sweeper.py` (new), `tests/cli_impl/test_idle_sweeper.py` (new)

- [ ] **A1.** Define `IdleSweeper`:
  ```python
  class IdleSweeper:
      def __init__(self, *, director, catalog_lookup, interval_seconds=30.0, stop_event=None): ...
      def tick(self) -> list[str]: ...     # returns evicted model_ids
      def start(self) -> threading.Thread: ...
      def stop(self) -> None: ...
  ```
  Inject `director` (LoadDirector) and `catalog_lookup` (Callable[[str], dict]) so unit tests don't need a real catalog.

- [ ] **A2.** `tick()` body per the spec's "Idle tick" pseudocode:
  - Snapshot `director.loaded` under `director.lock` (list of LoadEntry refs).
  - For each entry: skip if refcount > 0, idle_timeout missing, or last_touched too recent.
  - For each candidate: re-check under lock, pop, drop lock, call `disable_fn`, on failure re-insert. Append DecisionLogEntry under lock.
  - Return list of evicted model_ids.

- [ ] **A3.** Tests covering:
  - Picks up an idle candidate (refcount=0, last_touched > timeout).
  - Skips models with `idle_timeout_seconds is None`.
  - Skips models with `idle_timeout_seconds <= 0`.
  - Skips models with refcount > 0.
  - Skips models recently touched.
  - `disable_fn` failure: re-inserts LoadEntry, processes other candidates.
  - DecisionLogEntry recorded with `reason="idle_timeout:Ns"`.
  - catalog_lookup raises (model removed) → silently skipped.
  - Multiple candidates evicted in one tick.

- [ ] **A4.** Run tests, commit: `feat(idle_sweeper): per-model idle timeout sweeper + tests`.
