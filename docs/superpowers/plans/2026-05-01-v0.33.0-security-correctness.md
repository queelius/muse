## Task 2: SSRF protection + async fetch in image_input

**Finding:** `src/muse/modalities/image_generation/image_input.py:96-111` (`_fetch_http_url`) accepts any URL with no IP validation, fetches synchronously inside an async route. Allows attacks against link-local (AWS metadata), loopback (worker ports), and private LAN ranges. Also blocks the event loop for up to 30s per fetch.

**Files:**
- Modify: `src/muse/modalities/image_generation/image_input.py`
- Modify: `src/muse/modalities/image_generation/routes.py` (callers must await)
- Modify: `src/muse/modalities/image_embedding/routes.py` (caller must await)
- Test: `tests/modalities/image_generation/test_image_input.py`

**Steps:**
- [ ] Add `_validate_public_host(url) -> None` helper using `ipaddress.ip_address` to reject loopback/private/link-local/multicast/reserved on the resolved IP. Resolve via `socket.gethostbyname` (synchronous; cheap; happens once per fetch). Allow override via `MUSE_ALLOW_PRIVATE_FETCH=1` env var (operators on trusted networks).
- [ ] Convert `_fetch_http_url` to `async def` using `httpx.AsyncClient(timeout=_HTTP_TIMEOUT, follow_redirects=True)`.
- [ ] Convert `decode_image_input` to `async def` (only the http path needs await; data URL stays sync but the function becomes async for uniform await).
- [ ] Update callers in `image_generation/routes.py` and `image_embedding/routes.py` to `await decode_image_input(...)`. Use `asyncio.to_thread` for the data-URL path inside async route if the iteration count matters (it doesn't , data-URL decode is in-memory).
- [ ] Tests:
  - `test_fetch_blocks_loopback`: monkeypatch `socket.gethostbyname` to return `127.0.0.1`; assert ValueError "blocked".
  - `test_fetch_blocks_link_local`: returns `169.254.169.254`; assert ValueError.
  - `test_fetch_blocks_private`: returns `192.168.0.5`; assert ValueError.
  - `test_fetch_allows_override_via_env`: `MUSE_ALLOW_PRIVATE_FETCH=1`; loopback now succeeds (existing httpx mock).
  - Update existing `test_decode_http_url_fetches_via_httpx` to monkeypatch DNS to a public IP and use AsyncClient mock.
- [ ] Run `pytest tests/modalities/image_generation/test_image_input.py tests/modalities/image_embedding/ tests/modalities/image_generation/test_routes.py -q`.
- [ ] Commit: `fix(image_input): block SSRF to private/loopback/link-local; async fetch`

## Task 3: Bounded-read uploads in image_input

**Finding:** `decode_image_file` does `await file.read()` (no limit), checks size after. A 1GB upload is fully buffered before rejection. Used by `/v1/images/edits`, `/variations`, `/upscale`, `/segment`.

**Files:**
- Modify: `src/muse/modalities/image_generation/image_input.py`
- Test: `tests/modalities/image_generation/test_image_input.py`

**Steps:**
- [ ] In `decode_image_file`: change `raw = await file.read()` to `raw = await file.read(max_bytes + 1)`.
- [ ] In `_decode_data_url`: pre-check the b64 string length against `(max_bytes + 1) * 4 // 3 + 4` (b64 inflation factor) BEFORE calling `base64.b64decode`. Reject early on oversize.
- [ ] Update `_FakeUploadFile` in test file: `async def read(self, size: int | None = None)` , return only the first `size` bytes when given. Add a test confirming `read` is called with `max_bytes + 1`.
- [ ] New test `test_decode_image_file_rejects_oversize_without_full_read`: spy on read; assert read called with `max_bytes + 1`, oversize rejected.
- [ ] New test `test_decode_data_url_rejects_oversize_before_b64_decode`: build b64 string > cap; pass max_bytes; assert ValueError raised before `b64decode` runs (use `monkeypatch.setattr(image_input.base64, "b64decode", spy_that_raises)`; ValueError should fire first).
- [ ] Run `pytest tests/modalities/image_generation/test_image_input.py -q`.
- [ ] Commit: `fix(image_input): bounded read of multipart uploads and data URLs`

## Task 4: Streaming chat error envelope

**Finding:** `src/muse/modalities/chat_completion/routes.py:148-152` swallows producer-thread exceptions: logs error, then yields plain `[DONE]` (indistinguishable from clean finish). OpenAI's spec emits SSE `event: error` with the error envelope. Producer also uses `run_coroutine_threadsafe(...).result()` (blocking) instead of the `call_soon_threadsafe` pattern that audio_speech uses correctly.

**Files:**
- Modify: `src/muse/modalities/chat_completion/routes.py`
- Test: `tests/modalities/chat_completion/test_routes.py`

**Steps:**
- [ ] In `_producer`: replace `asyncio.run_coroutine_threadsafe(queue.put(...), loop).result()` with `loop.call_soon_threadsafe(queue.put_nowait, ...)` for all three call sites (chunk, exception, sentinel).
- [ ] In `_events`: when `isinstance(item, Exception)` is true, yield two events:
  ```python
  err_payload = {"error": {"code": "internal", "message": str(item), "type": "server_error"}}
  yield {"event": "error", "data": json.dumps(err_payload)}
  yield {"data": DONE_SENTINEL}
  return
  ```
- [ ] Add `import json` at top if not already.
- [ ] New test `test_stream_error_yields_error_event`: stub `model.chat_stream` to raise `RuntimeError("boom")`; collect all SSE events; assert one has `event="error"` with parsed JSON containing `error.message == "boom"`, followed by `[DONE]`.
- [ ] Run `pytest tests/modalities/chat_completion/test_routes.py -q`.
- [ ] Commit: `fix(chat): emit SSE error event on stream backend failure`

## Task 6: Pip subprocess timeouts

**Finding:** `src/muse/cli_impl/refresh.py:139,152`, `src/muse/core/venv.py:36,52`, `src/muse/core/install.py:26` all omit `timeout=` on `subprocess.run`. A hung mirror wedges the parent indefinitely. Probe and admin/operations already use `timeout=` (600s/1800s).

**Files:**
- Modify: `src/muse/cli_impl/refresh.py`
- Modify: `src/muse/core/venv.py`
- Modify: `src/muse/core/install.py`
- Test: `tests/cli_impl/test_refresh.py`

**Steps:**
- [ ] Add module-level constant `_PIP_TIMEOUT = 1800` (30 min) at top of each of the three files.
- [ ] In `refresh.py`: add `timeout=_PIP_TIMEOUT` to both `subprocess.run` calls. Wrap each in `try/except subprocess.TimeoutExpired` and return `RefreshResult(state="failed", message="pip install timed out after 30 min", extras=...)`.
- [ ] In `venv.py`: add `timeout=_PIP_TIMEOUT` to both `subprocess.run` calls. Let `TimeoutExpired` propagate (caller , `catalog.pull` , will surface as install failure).
- [ ] In `install.py`: add `timeout=_PIP_TIMEOUT` to the `subprocess.run` call. Let `TimeoutExpired` propagate.
- [ ] New test `test_refresh_returns_failed_on_pip_timeout`: monkeypatch `subprocess.run` to raise `TimeoutExpired(cmd, _PIP_TIMEOUT)`; call `refresh_one`; assert `state="failed"` and message mentions "timed out".
- [ ] Run `pytest tests/cli_impl/test_refresh.py tests/core/ -q`.
- [ ] Commit: `fix(pip): add 30-min timeout to all pip install subprocesses`
