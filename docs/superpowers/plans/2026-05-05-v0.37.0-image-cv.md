# v0.37.0 Implementation Plan: image/cv

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `image/cv` modality with three routes (`POST /v1/images/depth`, `/v1/images/keypoints`, `/v1/images/detect`), three runtimes, three bundled models (depth-anything-v2-small, vitpose-base-simple, detr-resnet-50), curated alternatives.

**Architecture:** One modality umbrella with three routes; capability flags (`supports_depth`, `supports_keypoints`, `supports_detection`) gate route acceptance per model. Three sibling runtimes wrap `AutoModelForDepthEstimation`, `AutoModelForKeypointDetection`, `AutoModelForObjectDetection`. One HF plugin with three-way dispatch.

**Tech Stack:** Existing. transformers >= 4.46 (for AutoModelForKeypointDetection), torch, PIL, numpy.

**Spec:** `docs/superpowers/specs/2026-05-05-v0.37.0-image-cv-design.md`.

## Task A: Protocol + codec + clients + skeleton

Three sibling protocols and result dataclasses; codec helpers for PNG16 + float32 + per-route envelope; three HTTP clients; modality __init__ exports + PROBE_DEFAULTS.

**Key codec functions:**

```python
def encode_depth_png16(depth: np.ndarray) -> tuple[bytes, float, float]:
    """Encode float depth map as 16-bit PNG. Returns (png_bytes, min, max)
    so the wire envelope can include the bounds."""

def encode_depth_float32(depth: np.ndarray) -> tuple[bytes, float, float]:
    """Raw float32 little-endian bytes + min/max bounds."""

def encode_depth_envelope(...) -> dict: ...
def encode_keypoints_envelope(...) -> dict: ...
def encode_detections_envelope(...) -> dict: ...
```

**Three result dataclasses:**

```python
@dataclass
class DepthResult:
    depth: np.ndarray  # (H, W) float
    model_id: str
    metric_depth: bool = False

@dataclass
class KeypointDetection:
    bbox: tuple[float, float, float, float]
    score: float
    keypoints: list[dict]  # [{name, x, y, score}, ...]

@dataclass
class KeypointResult:
    detections: list[KeypointDetection]
    model_id: str
    image_size: tuple[int, int]

@dataclass
class ObjectDetection:
    bbox: tuple[float, float, float, float]
    score: float
    label: str

@dataclass
class ObjectDetectionResult:
    detections: list[ObjectDetection]
    model_id: str
    image_size: tuple[int, int]
```

**Three Protocols:**

```python
class DepthEstimator(Protocol):
    def estimate_depth(self, image: Any) -> DepthResult: ...

class KeypointDetector(Protocol):
    def detect_keypoints(self, image: Any, *, threshold: float = 0.3) -> KeypointResult: ...

class ObjectDetector(Protocol):
    def detect_objects(self, image: Any, *, threshold: float = 0.5,
                       max_detections: int = 100) -> ObjectDetectionResult: ...
```

**Three clients (DepthClient, KeypointClient, ObjectDetectionClient):** mirror the OcrClient pattern (multipart upload, MUSE_SERVER fallback, raise_for_status).

**Tests:** Round-trip PNG16 encode/decode; float32 base64 round-trip; keypoint structure construction; detection sort+cap; client request body construction; client response parsing.

Commit: `feat(image/cv): protocol + codec + clients + modality skeleton`.

## Task B: HFDepthRuntime

Wraps `AutoModelForDepthEstimation` + `AutoImageProcessor`. Inference path:

```python
inputs = self._processor(image, return_tensors="pt").to(self._device)
outputs = self._model(**inputs)
depth = outputs.predicted_depth  # (1, H, W)
depth = F.interpolate(
    depth.unsqueeze(1), size=image.size[::-1], mode="bicubic"
).squeeze().cpu().numpy()
```

Capability flag `metric_depth: bool` (constructor arg, default False) decides post-processing: relative depth gets normalized to [0, 1]; metric depth stays in meters.

Tests: lazy-import discipline, returns `DepthResult`, metric vs relative path, image-size resampling.

Commit: `feat(image/cv): HFDepthRuntime`.

## Task C: HFKeypointRuntime

Wraps `AutoModelForKeypointDetection` + `AutoImageProcessor`. ViTPose-style: takes a single full-image bbox per call (the whole image). Inference:

```python
W, H = image.size
boxes = [[(0.0, 0.0, W, H)]]  # list of person-bboxes per image; here just one
inputs = self._processor(image, boxes=boxes, return_tensors="pt").to(self._device)
outputs = self._model(**inputs)
results = self._processor.post_process_keypoint_detection(
    outputs, target_sizes=[(H, W)], threshold=threshold,
)
# results[0] is a list of keypoint dicts
```

Some processors don't have `post_process_keypoint_detection`; runtime falls back to manual decoding (tensor to dict).

Tests: lazy-import discipline, returns `KeypointResult`, threshold filtering, name vs index fallback.

Commit: `feat(image/cv): HFKeypointRuntime`.

## Task D: HFObjectDetectionRuntime

Wraps `AutoModelForObjectDetection` + `AutoImageProcessor`:

```python
W, H = image.size
inputs = self._processor(images=image, return_tensors="pt").to(self._device)
outputs = self._model(**inputs)
results = self._processor.post_process_object_detection(
    outputs, target_sizes=[(H, W)], threshold=threshold,
)
# results[0] = {"scores": tensor, "labels": tensor, "boxes": tensor}
# Convert boxes from xyxy to xywh.
```

Sort by score desc, cap at `max_detections`, look up label names via `model.config.id2label`.

Tests: lazy-import discipline, returns `ObjectDetectionResult`, threshold filter, max_detections cap, label resolution.

Commit: `feat(image/cv): HFObjectDetectionRuntime`.
