# v0.38.0 Implementation Plan: audio/classification

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `audio/classification` modality with `POST /v1/audio/classifications`, single bundled AST AudioSet model, four curated alternatives. Wire shape mirrors `/v1/text/classifications`.

**Architecture:** Single runtime `HFAudioClassifier` over `AutoModelForAudioClassification` + `AutoFeatureExtractor` + librosa decode. Multipart upload (mirrors `/v1/audio/transcriptions`).

## Tasks (compact)

**A**: protocol + codec + client + skeleton. Uniform shape with text_classification's `ClassificationResult`-like dataclass. Multipart client mirrors AudioTranscriptionClient.

**B**: HFAudioClassifier runtime. Wraps `AutoModelForAudioClassification` + `AutoFeatureExtractor`. librosa.load decode at the model's sampling_rate. Softmax (default) vs sigmoid (when `problem_type == "multi_label_classification"`). Lazy imports (torch, transformers Auto*, librosa as module-level sentinels).

**C**: POST /v1/audio/classifications route. Multipart `file=` upload; size cap via `MUSE_AUDIO_CLS_MAX_BYTES` env (default 50MB). Saves to a temp file (librosa needs a path), runs inference, returns JSON envelope. Per-backend `_inference_lock` from v0.34.0.

**D**: Bundled `ast-audioset` script. `MIT/ast-finetuned-audioset-10-10-0.4593`, BSD-3-Clause, 527 AudioSet classes, ~340MB. Subclass of HFAudioClassifier.

**E**: HF plugin. Sniff `audio-classification` tag + repo-name fallbacks (`wav2vec2`, `ast-`, `hubert`, `mms-lid`, `panns`). Priority 110.

**F**: Curated entries (4): wav2vec2-emotion-en, hubert-superb-er, ast-speech-commands, mms-lid-126.

**G**: Docs + smoke matrix + CLAUDE.md modality bullet + conventions paragraph.

**H**: Release v0.38.0 (bump version, run full fast-lane suite, tag, push, GitHub release).

Estimated diff: ~1000-1300 LOC including tests.
