# docs/superpowers/

Historical session artifacts generated by the `superpowers:brainstorming` and `superpowers:writing-plans` skills. Each file under `specs/` or `plans/` corresponds to a specific release cycle (filename prefix is the date).

## NOT current architecture

These documents describe **original design intent** at the time of each release. Many of them describe approaches that were replaced or revised during implementation. Some describe features that were ultimately deferred or cancelled.

For **current** architecture, read in this order:
1. `CLAUDE.md` (repo root) -- the authoritative project narrative, kept current
2. `docs/CHAT_COMPLETION.md`, `docs/HF_PLUGINS.md`, `docs/MODEL_SCRIPTS.md`, `docs/RESOLVERS.md` -- current architecture references for specific subsystems

## Reading historical specs

Files that describe approaches superseded by later work carry an `OBSOLETE` or `REVISED IN IMPLEMENTATION` header at the top. The header names the specific change and points at the replacement. Files without such a header reflect what actually shipped.

## Note on install commands

Pre-v0.45.2, the PyPI distribution name was planned as `muse`. After v0.45.2 it became `museq` (the short name was squatted). Historical specs and plans contain `pip install muse[server]` references. The current install command is `pip install museq[server]`. The runtime code has been swept (v0.45.3); only the historical session docs retain the old form, intentionally, to preserve the record of what was true at the time of each release.

## Lifecycle

Future brainstorming + writing-plans skill invocations will continue to write here. After a release ships, the spec/plan becomes a historical artifact. The brainstorm process for the NEXT release may supersede earlier decisions; check the `OBSOLETE` headers (if any) on related specs before relying on past plans.
